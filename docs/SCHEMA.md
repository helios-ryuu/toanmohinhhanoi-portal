# Database Schema — Toán Mô Hình Hà Nội Portal

## 1. Tổng quan kiến trúc

Dự án sử dụng **Supabase** với ba thành phần chính:

| Thành phần | Mục đích |
|---|---|
| **Supabase Auth** (`auth.*`) | Quản lý danh tính người dùng qua Google OAuth. Không được chỉnh sửa trực tiếp. |
| **PostgreSQL Public** (`public.*`) | Toàn bộ dữ liệu nghiệp vụ của ứng dụng. |
| **Supabase Storage** | Lưu trữ file nộp bài (submission) và ảnh bìa bài viết. |

### Tương tác giữa `auth` và `public`

```
[Google OAuth]
     │
     ▼
auth.users  (managed by Supabase)
  - id (uuid)
  - email
  - raw_user_meta_data  ← chứa name, avatar_url từ Google
     │
     │  Trigger: trg_on_auth_user_created
     ▼
public.users  (application profile)
  - id (uuid, FK → auth.users.id)
  - username  (auto-generated từ email prefix)
  - display_name  (lấy từ Google metadata)
  - ...
```

Khi một người dùng đăng nhập Google lần đầu tiên, Supabase Auth tạo bản ghi trong `auth.users`. Trigger `trg_on_auth_user_created` lập tức tạo bản ghi tương ứng trong `public.users`, đảm bảo luôn tồn tại profile cho mọi người dùng xác thực.

## 2. Schema chi tiết

### 2.1 `public.users` — Profile người dùng

Mở rộng thông tin từ `auth.users`. Chỉ chứa dữ liệu văn bản; avatar lưu trong Storage hoặc dùng URL từ Google.

```sql
create table public.users (
    id            uuid        primary key references auth.users(id) on delete cascade,
    username      text        not null unique,
    display_name  text,
    bio           text,
    school        text,
    created_at    timestamptz not null default now(),
    updated_at    timestamptz,

    constraint users_username_len   check (char_length(username) between 3 and 30),
    constraint users_username_fmt   check (username ~ '^[a-z0-9_]+$')
);
```

| Cột | Kiểu | Ghi chú |
|---|---|---|
| `id` | `uuid` | **PK**, **FK** → `auth.users.id`. Cascade delete: xóa auth user thì xóa profile. |
| `username` | `text` | Unique. Auto-tạo từ phần đầu email khi đăng ký lần đầu. |
| `display_name` | `text` | Lấy từ `raw_user_meta_data->>'full_name'` của Google. |
| `bio` | `text` | Giới thiệu ngắn. |
| `school` | `text` | Trường học (hữu ích cho cuộc thi). |

**RLS Policies:**

```sql
alter table public.users enable row level security;

-- Mọi người có thể đọc profile
create policy "users: public read"
    on public.users for select
    using (true);

-- Chỉ chủ sở hữu mới được cập nhật
create policy "users: owner update"
    on public.users for update
    using (auth.uid() = id);
```

### 2.2 `public.post` — Bài viết / Tin tức

Bỏ `author_id` và `series_id`. Bài viết thuộc về tổ chức, không có tác giả cá nhân.

```sql
create table public.post (
    id           bigint      generated by default as identity primary key,
    slug         text        not null unique,
    title        text        not null,
    description  text        not null,
    content      text        not null,
    image_url    text,
    category     text        not null default 'news',
    published    boolean     not null default false,
    published_at timestamptz,
    created_at   timestamptz not null default now(),
    updated_at   timestamptz,

    constraint post_category_check check (
        category in ('news', 'announcement', 'tutorial', 'result')
    ),
    constraint post_publish_check check (
        (published = false and published_at is null) or
        (published = true  and published_at is not null)
    )
);
```

| Cột | Kiểu | Ghi chú |
|---|---|---|
| `id` | `bigint` | **PK** |
| `slug` | `text` | Unique. Dùng cho URL. |
| `category` | `text` | Phân loại: `news`, `announcement`, `tutorial`, `result`. |
| `image_url` | `text` | URL ảnh bìa (từ Supabase Storage bucket `post-images`). |

**RLS Policies:**

```sql
alter table public.post enable row level security;

-- Bài đã publish thì public đọc được
create policy "post: public read published"
    on public.post for select
    using (published = true);

-- Chỉ service role (admin server-side) được write
-- → Không có policy insert/update/delete cho anon/authenticated
-- → Mọi thao tác ghi đi qua API route với service_role key
```

### 2.5 `public.contest` — Cuộc thi (v0.2.0 timeline model)

Từ v0.2.0, lịch trình cuộc thi gồm hai lớp:

* **Khung tổng** (`start_at` / `end_at`): biên thời gian của cả cuộc thi.
* **Các giai đoạn động** (`contest_stage`): n giai đoạn, có thể chồng chéo, mỗi giai đoạn độc lập bật `allow_registration` / `allow_submission`.

```sql
create table public.contest (
    id                  bigint      generated by default as identity primary key,
    slug                text        not null unique,
    title               text        not null,
    description         text        not null,
    rules               text,
    cover_image_url     text,
    participation_type  text        not null default 'individual',
    max_team_size       smallint    not null default 1,
    start_at            timestamptz not null,
    end_at              timestamptz not null,
    status              text        not null default 'draft',
    created_at          timestamptz not null default now(),
    updated_at          timestamptz,

    constraint contest_participation_type_check check (
        participation_type in ('individual', 'team', 'both')
    ),
    constraint contest_status_check check (
        status in ('draft', 'active', 'closed', 'cancelled')
    ),
    constraint contest_team_size_check check (
        (participation_type = 'individual' and max_team_size = 1) or
        (participation_type in ('team', 'both') and max_team_size >= 2)
    ),
    constraint contest_grand_order check (start_at < end_at)
);
```

| Cột | Kiểu | Ghi chú |
|---|---|---|
| `participation_type` | `text` | `individual` (chỉ cá nhân), `team` (chỉ đội), `both` (cả hai). |
| `max_team_size` | `smallint` | Số thành viên tối đa mỗi đội. Bằng 1 khi `individual`. |
| `start_at` / `end_at` | `timestamptz` | Khung thời gian tổng của cuộc thi. Mọi giai đoạn phải nằm trong khoảng này. |
| `status` | `text` | `draft \| active \| closed \| cancelled`. |

#### `public.contest_stage` — Giai đoạn động

```sql
create table public.contest_stage (
    id                 bigint      generated by default as identity primary key,
    contest_id         bigint      not null references public.contest(id) on delete cascade,
    name               text        not null,
    description        text,
    start_at           timestamptz not null,
    end_at             timestamptz not null,
    allow_registration boolean     not null default false,
    allow_submission   boolean     not null default false,
    submission_type    text,
    display_order      integer     not null default 0,
    created_at         timestamptz not null default now(),
    updated_at         timestamptz,
    constraint contest_stage_order_check check (start_at < end_at)
);
```

| Cột | Ghi chú |
|---|---|
| `allow_registration` | Khi `now ∈ [start_at, end_at]` và cờ này bật, đăng ký được mở. |
| `allow_submission` | Tương tự cho việc nộp bài. |
| `submission_type` | Gợi ý loại bài nộp (file/quiz/link…). |
| `display_order` | Thứ tự sắp xếp trên UI khi liệt kê các giai đoạn. |

**RLS Policies:**

```sql
alter table public.contest enable row level security;

create policy "contest: public read non-draft"
    on public.contest for select
    using (status <> 'draft');
```

### 2.6 `public.contest_registration` — Đăng ký tham gia

Một bản ghi đại diện cho một đội/cá nhân đã đăng ký. Với cuộc thi cá nhân, đội chỉ có một thành viên.

```sql
create table public.contest_registration (
    id           bigint      generated by default as identity primary key,
    contest_id   bigint      not null references public.contest(id) on delete cascade,
    team_name    text,
    status       text        not null default 'pending',
    registered_at timestamptz not null default now(),
    updated_at   timestamptz,

    constraint registration_status_check check (
        status in ('pending', 'approved', 'rejected', 'withdrawn')
    )
);
```

| Cột | Ghi chú |
|---|---|
| `team_name` | Tên đội. Với cuộc thi cá nhân, để `null` hoặc dùng `display_name`. |
| `status` | `pending` → chờ duyệt; `approved` → đã chấp nhận; `rejected`/`withdrawn` → không tham gia. |

### 2.7 `public.registration_member` — Thành viên trong đăng ký

Tách ra khỏi `contest_registration` để chuẩn hóa quan hệ nhiều-nhiều giữa đăng ký và người dùng.

```sql
create table public.registration_member (
    registration_id bigint not null references public.contest_registration(id) on delete cascade,
    user_id         uuid   not null references public.users(id) on delete cascade,
    role            text   not null default 'member',
    joined_at       timestamptz not null default now(),
    primary key (registration_id, user_id),

    constraint member_role_check check (role in ('leader', 'member'))
);
```

| Cột | Ghi chú |
|---|---|
| `user_id` | **FK** → `public.users.id`. |
| `role` | `leader` — người đại diện đội; `member` — thành viên thường. |

**Ràng buộc nghiệp vụ** (thực thi qua trigger hoặc application layer):
- Mỗi đăng ký có đúng một `leader`.
- Số `member` ≤ `contest.max_team_size`.
- Một user không thể đăng ký nhiều hơn một lần cho cùng một cuộc thi.

**RLS Policies:**

```sql
alter table public.contest_registration enable row level security;
alter table public.registration_member enable row level security;

-- Thành viên xem được đăng ký của mình
create policy "registration: member read own"
    on public.contest_registration for select
    using (
        id in (
            select registration_id from public.registration_member
            where user_id = auth.uid()
        )
    );

create policy "registration_member: member read own"
    on public.registration_member for select
    using (user_id = auth.uid());
```

### 2.8 `public.submission` — Bài nộp

```sql
create table public.submission (
    id              bigint      generated by default as identity primary key,
    registration_id bigint      not null references public.contest_registration(id) on delete cascade,
    storage_path    text        not null,
    file_name       text        not null,
    file_size_bytes bigint      not null,
    mime_type       text        not null,
    note            text,
    submitted_by    uuid        not null references public.users(id),
    submitted_at    timestamptz not null default now(),
    is_final        boolean     not null default false,

    constraint submission_file_size_check check (file_size_bytes > 0)
);
```

| Cột | Ghi chú |
|---|---|
| `registration_id` | **FK** — bài nộp thuộc về một đăng ký cụ thể. |
| `storage_path` | Đường dẫn đầy đủ trong Supabase Storage, ví dụ `submissions/{contest_id}/{registration_id}/{uuid}.pdf`. |
| `file_name` | Tên file gốc do người dùng upload. |
| `mime_type` | Định dạng file (`application/pdf`, `application/zip`, v.v.). |
| `submitted_by` | **FK** — người dùng thực hiện upload (phải là thành viên của đăng ký). |
| `is_final` | Đánh dấu đây là bản nộp chính thức cuối cùng. |

**RLS Policies:**

```sql
alter table public.submission enable row level security;

-- Thành viên đăng ký xem bài nộp của nhóm mình
create policy "submission: member read own"
    on public.submission for select
    using (
        registration_id in (
            select registration_id from public.registration_member
            where user_id = auth.uid()
        )
    );

-- Thành viên đăng ký được tạo bài nộp
create policy "submission: member insert"
    on public.submission for insert
    with check (
        submitted_by = auth.uid() and
        registration_id in (
            select registration_id from public.registration_member
            where user_id = auth.uid()
        )
    );
```

## 3. Triggers & Functions

### 3.1 Trigger: Tạo `public.users` khi Google đăng nhập lần đầu

```sql
create or replace function public.handle_new_auth_user()
returns trigger
language plpgsql
security definer set search_path = public
as $$
declare
    base_username text;
    final_username text;
    counter       int := 0;
begin
    -- Lấy phần đầu email làm username gốc, chỉ giữ ký tự a-z, 0-9, _
    base_username := lower(regexp_replace(split_part(new.email, '@', 1), '[^a-z0-9_]', '_', 'g'));

    -- Đảm bảo độ dài tối thiểu 3 ký tự
    if char_length(base_username) < 3 then
        base_username := base_username || '_usr';
    end if;

    final_username := base_username;

    -- Xử lý trùng username
    while exists (select 1 from public.users where username = final_username) loop
        counter := counter + 1;
        final_username := base_username || counter::text;
    end loop;

    insert into public.users (id, username, display_name)
    values (
        new.id,
        final_username,
        coalesce(new.raw_user_meta_data->>'full_name', new.raw_user_meta_data->>'name')
    );

    return new;
end;
$$;

-- Chỉ kích hoạt khi provider là Google (hoặc tổng quát hơn: mọi lần tạo user mới)
create trigger trg_on_auth_user_created
    after insert on auth.users
    for each row
    execute function public.handle_new_auth_user();
```

### 3.2 Trigger: Cập nhật `updated_at` tự động

```sql
create or replace function public.set_updated_at()
returns trigger
language plpgsql
as $$
begin
    new.updated_at = now();
    return new;
end;
$$;

-- Áp dụng cho các bảng có updated_at
create trigger trg_users_updated_at
    before update on public.users
    for each row execute function public.set_updated_at();

create trigger trg_post_updated_at
    before update on public.post
    for each row execute function public.set_updated_at();

create trigger trg_contest_updated_at
    before update on public.contest
    for each row execute function public.set_updated_at();

create trigger trg_registration_updated_at
    before update on public.contest_registration
    for each row execute function public.set_updated_at();
```

## 5. Supabase Storage — Cấu trúc Buckets

### Bucket: `post-images` (public)

Lưu ảnh bìa bài viết. Truy cập công khai qua URL.

```
post-images/
└── {post-slug}/
    └── cover.{ext}
```

**Storage Policy:**
- `select`: `true` (public read)
- `insert/update/delete`: chỉ service role (quản lý qua admin API)

---

### Bucket: `submissions` (private)

Lưu file bài nộp cuộc thi. Chỉ thành viên đăng ký và admin mới truy cập được.

```
submissions/
└── {contest_id}/
    └── {registration_id}/
        └── {submission_id}_{original_filename}
```

**Quy ước đặt tên file:** `{submission.id}_{file_name}` để tránh trùng lặp và dễ tra cứu.

**Storage RLS Policy (Supabase Storage):**

```sql
-- Thành viên đọc file của nhóm mình
create policy "submissions: member read own"
    on storage.objects for select
    using (
        bucket_id = 'submissions' and
        -- Trích contest_id và registration_id từ path
        (storage.foldername(name))[2]::bigint in (
            select registration_id from public.registration_member
            where user_id = auth.uid()
        )
    );

-- Thành viên upload file cho nhóm mình
create policy "submissions: member insert"
    on storage.objects for insert
    with check (
        bucket_id = 'submissions' and
        (storage.foldername(name))[2]::bigint in (
            select registration_id from public.registration_member
            where user_id = auth.uid()
        )
    );
```

## 8. Ghi chú di chuyển (Migration Notes)

- **Bỏ bảng `author`**: Xóa cột `author_id` khỏi `post`, xóa bảng `author`.
- **Bỏ bảng `series`**: Xóa cột `series_id`, `series_order`, `type` khỏi `post`, xóa bảng `series`.
- **Thay `public."user"` (auth tự xây)**: Xóa bảng `user` cũ (có `password_hash`), tạo bảng `users` mới liên kết với `auth.users`.
- **Chuyển sang Supabase Auth**: Xóa các API route `/api/auth/register`, `/api/auth/login`, `/api/auth/logout` tự xây; thay bằng Supabase Auth helpers.
